# 페이지 설계

## 페이지 구조

```
/                    → 메인 랜딩 페이지
/services            → 서비스 목록 (카테고리 필터, 검색)
/services/[slug]     → 서비스 상세
/admin               → 관리 대시보드
/admin/services      → 서비스 관리
/admin/containers    → Docker 컨테이너
/admin/logs          → 실시간 로그
/admin/system        → 시스템 모니터링
```

---

## 1. 메인 랜딩 페이지 (`/`)

### 구성 요소

1. **Hero 섹션**
   - 타이틀 및 서브타이틀
   - 실행 중인 서비스 수 표시
   - CTA 버튼 (Services, Admin)

2. **Service Categories 섹션**
   - AI, n8n, Infrastructure 3개 카테고리
   - 각 카테고리별 서비스 수 및 실행 상태

3. **Featured Services 섹션**
   - 현재 실행 중인 주요 서비스 카드

### 컴포넌트

- `Hero.tsx`: 히어로 배너
- `ServiceCategories.tsx`: 카테고리 카드 그리드
- `FeaturedServices.tsx`: 주요 서비스 목록

---

## 2. 서비스 목록 페이지 (`/services`)

### 기능

- **카테고리 필터**: All, AI, n8n, Infra
- **상태 필터**: Running only 토글
- **검색**: 이름, 설명, 기술스택 검색
- **그리드 뷰**: 반응형 카드 레이아웃

### 컴포넌트

- `ServiceCard.tsx`: 서비스 카드
- `StatusBadge.tsx`: 상태 표시기
- `TechBadge.tsx`: 기술스택 배지

---

```
├────────────────────────────────────────┤
│ Features                               │
│ • Feature 1                            │
│ • Feature 2                            │
├────────────────────────────────────────┤
│ Containers                             │
│ [container-1] [container-2]            │

```

### SSG (Static Site Generation)

```typescript
export async function generateStaticParams() {
  return services.map((service) => ({
    slug: service.id,
  }));
}
```

## 4. 관리 대시보드 (`/admin`)

### 레이아웃

```
┌──────────────┬──────────────────────────────────────┐
│              │                                      │
│  [Sidebar]   │      Main Content Area              │
│              │                                      │
│  Dashboard   │  ┌─────────────────────────────┐    │
│  Services    │  │  System Stats Cards         │    │
│  Containers  │  └─────────────────────────────┘    │
│  Logs        │                                      │
│  System      │  ┌─────────────────────────────┐    │
│              │  │  Running Services List      │    │
│              │  └─────────────────────────────┘    │
│              │                                      │
└──────────────┴──────────────────────────────────────┘
```

### 컴포넌트

- `Sidebar.tsx`: 데스크톱 사이드바
- `MobileSidebar.tsx`: 모바일 탭 네비게이션

---

## 5. 서비스 관리 (`/admin/services`)

### 기능

- 서비스 목록 (필터: all/running/stopped)
- 각 서비스별 시작/중지/재시작 버튼
- 외부 링크 열기

---

## 6. 컨테이너 관리 (`/admin/containers`)

### 기능

- 서비스별 컨테이너 그룹화
- 개별 컨테이너 시작/중지/재시작
- 상태 표시 (running/stopped)

---

## 7. 로그 조회 (`/admin/logs`)

### 기능

- 컨테이너 선택
- 로그 라인 수 설정 (50/100/500)
- Auto-refresh 토글
- 로그 다운로드/클리어

---

## 8. 시스템 모니터링 (`/admin/system`)

### 표시 정보

- CPU 사용률 (프로그레스 바)
- 메모리 사용량
- 디스크 사용량
- 시스템 Uptime
