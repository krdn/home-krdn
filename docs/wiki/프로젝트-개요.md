# 프로젝트 개요

## 배경

이 프로젝트는 시스템에서 운영 중인 다양한 서비스들을 한 곳에서 소개하고 관리할 수 있는 통합 플랫폼으로 기획되었습니다.

## 목적

1. **홍보**: 개발된 서비스들을 외부에 소개
2. **소개**: 각 서비스의 기능, 기술스택, 특징을 상세히 설명
3. **관리**: 서비스 상태 모니터링 및 제어

## 요구사항

### 핵심 요구사항

| 항목 | 선택 |
|------|------|
| 목적 | 혼합 (관리 + 홍보) |
| 서비스 범위 | 모든 프로젝트 (11개) |
| 디자인 | 테크 스타일 + 다크모드 기본 |
| 관리 기능 | 풀 관리 (시작/중지, 로그, 모니터링) |
| 기술 스택 | Next.js 15 |
| 인증 | 전체 공개 |
| 배포 | Docker (포트 8082) |

## 대상 사용자

- **개발자**: 서비스 관리 및 모니터링
- **외부 방문자**: 프로젝트 포트폴리오 열람

## 타임라인

프로젝트는 AI 어시스턴트와의 인터뷰 방식으로 진행되었으며, 다음 단계로 구현되었습니다:

1. 프로젝트 초기화
2. UI 컴포넌트 개발
3. 랜딩 페이지 구현
4. 서비스 페이지 구현
5. 관리 대시보드 구현
6. Docker 배포 설정
